# Тестовое задание: интеграция с Яндекс.Картами

## Описание
SPA-приложение на **Laravel 11** (бэкенд, API) и **Vue 3** (фронтенд) для сбора и отображения отзывов организаций с Яндекс.Карт.  
Реализована авторизация через **Sanctum**, добавление компаний, вывод основной информации (название, рейтинг, количество отзывов) и пагинация отзывов (структура готова).

## Стек технологий
- **Backend**: Laravel 11, PHP 8.3, Sanctum (SPA-аутентификация), MySQL, Eloquent ORM
- **Frontend**: Vue 3, Composition API, Vue Router, Axios, Vite
- **Инструменты**: Composer, NPM, Git

## Установка и запуск

### Требования
- PHP >= 8.3
- Composer
- Node.js >= 18
- MySQL (или другая СУБД)

Использование
Авторизация
Перейдите на главную страницу (/), введите логин и пароль, указанные выше.

Добавление компании
После входа вы попадёте на страницу /companies. Вставьте ссылку на организацию с Яндекс.Карт (например, https://yandex.ru/maps/-/CTEjzZJa) и нажмите «Сохранить».
Компания появится в списке.

Просмотр деталей
Нажмите «Смотреть отзывы» на карточке компании. Вы увидите название, рейтинг и количество отзывов (сейчас отображаются тестовые данные, так как полноценный парсинг отзывов требует доработки — см. раздел «Парсинг»).

Парсинг
В данном проекте парсинг данных с Яндекс.Карт реализован в сервисе app/Services/YandexParserService.php.

Текущие ограничения:

Яндекс.Карты используют динамическую подгрузку отзывов при прокрутке и не предоставляют официального API.

Для получения всех отзывов (до ~600) требуется headless-браузер (например, Puppeteer), который будет прокручивать страницу и извлекать данные.

Из-за ограниченного времени на выполнение задания в текущей версии используется заглушка — компания сохраняется с тестовыми данными (рейтинг 4.5, 123 отзыва), чтобы продемонстрировать работу интерфейса.

Код сервиса содержит комментарий @TODO, указывающий на необходимость доработки парсинга.

Предлагаемый подход для доработки:

Запустить отдельный микросервис на Node.js с Puppeteer.

Либо использовать php-webdriver (Selenium) для управления браузером из PHP.

Результат парсинга кэшировать в БД, чтобы не перегружать Яндекс и ускорить ответ.

Структура БД
companies — организации:
id, url (уникальная), yandex_id, name, rating, rating_count, review_count, last_parsed_at, created_at, updated_at

reviews — отзывы:
id, company_id (внешний ключ), author, date, text, rating, created_at, updated_at

Связь: один-ко-многим (Company → Review).

API Эндпоинты
Метод	URL	Описание
POST	/api/login	Авторизация (email + password)
GET	/api/user	Получить текущего пользователя
POST	/api/logout	Выйти
GET	/api/companies	Список всех компаний
POST	/api/companies	Создать новую компанию (запустить парсинг)
GET	/api/companies/{id}	Получить компанию с пагинированными отзывами
Все маршруты (кроме /login) защищены Sanctum, требуется токен в заголовке Authorization: Bearer {token}.

Что можно улучшить при наличии времени
Реализовать полноценный парсинг отзывов через Puppeteer или Selenium.

Добавить фоновую обработку парсинга через очереди (Laravel Queue), чтобы не блокировать ответ.

Кэшировать результаты парсинга в Redis (с TTL).

Написать юнит-тесты и feature-тесты для API и сервисов.

Улучшить интерфейс: индикаторы загрузки, анимации, более детальные сообщения об ошибках.

Добавить возможность обновлять данные по кнопке «Обновить».
